使用说明

SQLite是一种轻量级的关系型数据库，不需要独立的服务器进程或系统配置。您可以使用以下工具打开和查询这些数据库文件：

  • DB Browser for SQLite（免费图形界面工具）
  • Python sqlite3 模块（编程方式访问）
  • DBeaver（通用数据库管理工具）
  • Tableau / Power BI（可视化分析）

示例查询（Python）：

import sqlite3
conn = sqlite3.connect('youyang_zazu.db')
cursor = conn.cursor()
cursor.execute("SELECT * FROM paragraphs LIMIT 5")
for row in cursor.fetchall():
    print(row)
conn.close()

数据表结构

以下是"清洗数据"数据库中各数据表的结构说明：

paragraphs 表

存储原始文本段落数据，是数据库的基础表。

字段类型说明
idINTEGER主键
volumeTEXT所属卷目
contentTEXT段落原文
titleTEXT段落标题/摘要

shengui 表

存储神怪相关的分类标注数据。

字段类型说明
idINTEGER主键
paragraph_idINTEGER关联paragraphs.id
categoryTEXT神怪分类
tagsTEXT标签（JSON格式）

theme_details 表

存储主题详细分类信息。

字段类型说明
idINTEGER主键
paragraph_idINTEGER关联paragraphs.id
themeTEXT主题名称
sub_themeTEXT子主题

narrative_stats 表

存储叙事结构分类的统计结果。

monster_type_stats 表

存储妖怪类型分类的统计结果。

theme_frequency 表

存储主题频次统计数据。

数据来源分布

数据流程：

  1. OCR生文本：两份独立OCR处理的原始文本
  2. 文本去重：比对两份OCR结果，取重合部分为"真文本"
  3. 注释分离：识别注释格式，分离正文与注释
  4. 文本分段：参考注释，对正文进行精确分段
  5. LLM分类：依据分类细则，由LLM完成自动分类
  6. 人工校验：对分类结果进行抽样检查与修正
  7. 数据入库：导入SQLite数据库，建立关系表
