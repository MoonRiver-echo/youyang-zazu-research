# 《酉阳杂俎》数字人文研究项目汇报

## 一、项目简介

本项目以唐代段成式所撰笔记小说《酉阳杂俎》为研究对象，通过数字化手段进行文本整理、分类分析与可视化展示，探索唐代神怪宗教文化的数字人文研究方法。

### 研究内容
- **文本校勘**：对《酉阳杂俎》进行初步校勘，建立标准化数字文本
- **主题分类**：对全书条目进行叙事特征与主题分类统计
- **国别整理**：梳理书中涉及的域外国家与地理信息
- **可视化展示**：构建交互式数字人文网站展示研究成果

### 项目成果
- 生成标准数字化文本 1 份（酉阳杂俎-初校.md）
- 生成分类统计表格 20+ 份（CSV格式）
- 构建关系型数据库 1 套（SQLite）
- 开发可视化网站 1 个（前端交互展示）

## 三、研究过程中遇到的挑战

### 3.1 文献获取与处理方面的挑战

**挑战1：多版本文本的差异处理**
- 原始OCR文本存在大量识别错误（约15-20%错误率）
- 两个校本（方南生、许逸民）在条目划分、校勘体例上存在差异
- 三版本之间的对应关系难以自动建立

**挑战2：校勘记的自动提取**
- 许逸民校本注释分散在条后，格式不统一
- 方南生校本校勘记以"校勘记"标题开始，需要准确识别
- 注释跨行现象普遍，难以用简单规则提取

**挑战3：条目边界的自动识别**
- 原文条目无明确标记，需根据文本特征判断
- 不同版本对条目划分存在分歧
- 部分条目存在嵌套或引用关系

### 3.2 分类与标注方面的挑战

**挑战4：主题分类标准的确立**
- 《酉阳杂俎》内容庞杂，涵盖志怪、博物、方术、宗教等多领域
- 传统四部分类法难以适用
- 需要建立符合现代学科体系的新分类标准

**挑战5：人物与地名的识别**
- 书中人物众多，僧道方士难以归类
- 国名、地名古今异名，如"波斯"、"拂林"等需考证
- 部分词条兼具人名、地名、物名多重属性

**挑战6：注释术语的语义理解**
- 许校本注释中包含大量术语解释
- 需要根据注释内容判断词条属性
- 注释与正文之间存在复杂的指涉关系

### 3.3 技术实现方面的挑战

**挑战7：繁简转换的准确性**
- 古籍中大量异体字、通假字
- 繁简转换存在一对多问题
- 部分字形在Unicode中无对应

**挑战8：文本比对的算法设计**
- 三版本文字差异度难以量化
- 需要设计合适的相似度阈值
- 跨段落匹配需要处理位移问题

**挑战9：数据结构的规范化**
- 分类结果需要支持多重归类
- 条目与分类之间是多对多关系
- 需要设计支持统计分析的数据库结构

### 4.1 文本处理方案

**多版本对齐策略**
```python
# 核心思路：以OCR文本为锚点，建立三版本映射
# 使用SequenceMatcher计算文本相似度
# 设计分层匹配算法：先匹配卷首，再匹配条目

def line_match_score(block_text: str, candidate_line: str) -> float:
    # 提取前缀进行快速匹配
    block_prefix = normalize(block_text)[:28]
    line_prefix = normalize(candidate_line)[:28]
    # 结合精确匹配率与模糊匹配率
    ratio = SequenceMatcher(None, candidate_prefix, block_prefix).ratio()
    return max(common / max(4, min(len(block_prefix), len(line_prefix))), ratio * 0.7)
```
